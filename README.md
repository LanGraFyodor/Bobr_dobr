# Полет вслепую

Прототип резервной навигации для БПЛА и легких воздушных судов при отказе ГНСС.

Система берет цифровую модель рельефа `DEM` и поток радиовысотомера `NMEA-0183`, восстанавливает профиль рельефа под воздушным судном и находит на карте наиболее вероятную траекторию полета: координаты `X/Y`, скорость, курс, вектор скорости и уверенность результата.

## Ключевая идея

По ТЗ барометрическая высота полета известна:

```text
H_baro = 1500 м
```

Радиовысотомер измеряет высоту над землей. Значит, из NMEA можно восстановить абсолютный профиль рельефа:

```text
H_terrain = H_baro - H_radio
```

Дальше задача становится задачей terrain matching: найти на DEM такую прямую траекторию, чей профиль высот лучше всего совпадает с восстановленным профилем радиовысотомера.

## Что умеет проект

- читает DEM-карты `.tif` через `rasterio`;
- переводит координаты карты из `lat/lon` в UTM-метры;
- парсит NMEA `$GPGGA`;
- генерирует тестовые NMEA-файлы по реальной DEM;
- ищет стартовые координаты `X/Y`, текущие координаты, скорость и азимут;
- считает вектор скорости `Vx/Vy`;
- сравнивает профили по `RMSE` и Pearson correlation;
- строит heatmap корреляции по направлениям `0-359°`;
- накладывает найденную траекторию на карту высот;
- показывает confidence и оценку точности в метрах;
- умеет работать с плоскими/слабоинформативными участками;
- сглаживает траекторию фильтром Калмана или скользящим средним;
- ускоряет тяжелый перебор гипотез Rust-ядром;
- дает демонстрационный Streamlit-интерфейс.

## Алгоритмы

### RMSE

Базовый режим. Перебирает гипотезы:

```text
стартовая точка X/Y + скорость + азимут
```

Для каждой гипотезы строится прямая траектория, с DEM снимается профиль высот, затем считается ошибка:

```text
RMSE = sqrt(mean((DEM_profile - NMEA_profile)^2))
```

Побеждает гипотеза с минимальной ошибкой и высокой корреляцией формы профиля.

### Weighted scoring

Режим для сложных данных. Он сильнее учитывает информативные участки профиля: перепады, изгибы и выраженные особенности рельефа. Плоские фрагменты получают меньший вес, потому что по ним хуже привязываться к карте.

### Bridge mode

Режим продолжения траектории. Если в профиле есть слабый/плоский участок, алгоритм берет уверенный фрагмент до него, определяет по нему курс и скорость, затем продолжает единую прямую траекторию через неинформативную зону. Это полезно для сценариев вроде озера, моря, ровной тундры или участка с бедным рельефом.

### Coarse-to-fine search

Чтобы не считать всю карту с максимальной плотностью, поиск идет каскадом:

1. грубый поиск по карте;
2. выбор лучших кандидатов;
3. уточнение вокруг лучших стартовых точек;
4. micro/nano refine;
5. финальная проверка профиля.

### Rust-core

Самый тяжелый расчет вынесен в Rust:

- перебор стартовых точек;
- перебор скоростей;
- перебор азимутов;
- bilinear sampling DEM;
- RMSE;
- correlation;
- weighted scoring;
- раннее отсечение слабых кандидатов.

Python управляет пайплайном и интерфейсом, Rust считает тяжелую математику.

## Запуск

```powershell
pip install -r requirements.txt
python -m streamlit run app.py --server.address 127.0.0.1 --server.port 8501
```

Открыть:

```text
http://127.0.0.1:8501
```

Для Rust-ускорения нужен установленный Rust toolchain. При необходимости библиотека собирается из `rust/terrain_nav_core`.

## Быстрый сценарий проверки

1. Оставить стандартную карту `data/map.tif` или загрузить свою DEM-карту.
2. Загрузить NMEA или нажать `Сгенерировать NMEA`.
3. Выбрать пресет поиска: `Быстро`, `Точно` или `Максимально`.
4. Выбрать алгоритм: `RMSE`, `Weighted scoring`, `Bridge mode`.
5. Нажать `Запустить локализацию`.
6. Проверить метрики, heatmap, профиль высот и траекторию на DEM.

## Выходные данные

Интерфейс показывает:

- скорость, м/с;
- путевой курс, градусы;
- `Vx/Vy`, м/с;
- стартовые координаты `X/Y`;
- текущие координаты `X/Y`;
- `RMSE`;
- correlation;
- confidence;
- оценку точности;
- карту траектории;
- сравнение профиля NMEA и профиля DEM.

## Что закрыто по ТЗ

- преобразование радиовысоты в профиль абсолютных высот рельефа;
- корреляционный поиск по направлениям;
- перебор азимутов `0-359°`;
- расчет скорости, курса и вектора движения;
- географическая привязка `X/Y`;
- heatmap коэффициентов корреляции;
- наложение найденной траектории на карту высот;
- генерация NMEA в формате `$GPGGA`;
- самооценка точности;
- адаптация к плоскому рельефу;
- сглаживание;
- оптимизация для маломощного вычислителя через Rust-core.

## Ограничения прототипа

- основная модель предполагает прямолинейный равномерный участок полета;
- барометрическая высота по умолчанию считается постоянной `1500 м`;
- на полностью плоском рельефе один радиовысотомер не дает уникальной геопривязки;
- низкий confidence означает не сбой, а честный сигнал: похожих маршрутов много или профиль слабо информативен;
- это хакатонный прототип, не сертифицированная авиационная система.
