# Полет вслепую

Веб-прототип резервной навигации без ГНСС.  
По цифровой модели рельефа `DEM` и сообщениям радиовысотомера `NMEA-0183` проект оценивает положение воздушного судна, скорость, курс и вектор движения.

## Идея

Радиовысотомер дает высоту над землей. По ТЗ барометрическая высота полета постоянная:

```text
H_baro = 1500 м
H_terrain = H_baro - H_radio
```

Так из NMEA восстанавливается профиль абсолютных высот рельефа под самолетом. Затем алгоритм ищет на DEM прямую траекторию, профиль которой максимально похож на восстановленный профиль.

## Что реализовано

- чтение `map.tif` через `rasterio`;
- перевод карты из `lat/lon` в UTM-метры через `pyproj`;
- парсер NMEA `$GPGGA`;
- генератор тестовых NMEA-данных по реальной DEM-карте;
- поиск стартовых координат `x/y`, текущих координат, скорости и азимута;
- расчет вектора скорости `Vx/Vy`;
- перебор гипотез `стартовая точка + скорость + азимут`;
- bilinear-сэмплинг высот DEM вдоль траектории;
- основной поиск по `RMSE`;
- режим `Weighted scoring` для усиления информативных перепадов рельефа;
- расчет Pearson correlation профилей;
- полная heatmap корреляции по направлениям `0..359°`;
- наложение найденной траектории на DEM;
- сравнение профиля NMEA и профиля DEM;
- confidence и оценка точности в метрах;
- проверка плоского/неинформативного рельефа;
- сглаживание траектории фильтром Калмана или скользящим средним;
- ускоренное ядро на Rust для массового перебора гипотез;
- Streamlit-интерфейс для демонстрации.

## Стек

```text
Python, Streamlit, Rasterio, PyProj, NumPy, SciPy, Matplotlib
Rust, Cargo, ctypes
```

Python отвечает за интерфейс, чтение файлов, подготовку данных и визуализацию.  
Rust отвечает за самый тяжелый расчет: перебор траекторий, bilinear-интерполяцию DEM, RMSE, Weighted RMSE и correlation.

## Запуск

```bash
pip install -r requirements.txt
python -m streamlit run app.py --server.address 127.0.0.1 --server.port 8501
```

Открыть:

```text
http://127.0.0.1:8501
```

Окно терминала закрывать нельзя: пока оно открыто, работает веб-приложение.  
Если команда `python` не находится, используйте Python из вашей IDE/виртуального окружения или укажите полный путь к интерпретатору.

Для Rust-ускорения нужен установленный Rust toolchain (`cargo`). При первом запуске библиотека собирается автоматически из `rust/terrain_nav_core`.

## Быстрый сценарий

1. Оставить стандартную карту `data/map.tif` или загрузить свой `.tif`.
2. Загрузить NMEA или нажать `Сгенерировать NMEA`.
3. Выбрать алгоритм: `RMSE`, `Weighted scoring`, `Bridge mode`.
4. Нажать `Запустить локализацию`.
5. Смотреть:
   - скорость;
   - курс;
   - `Vx/Vy`;
   - стартовые и текущие `X/Y`;
   - `RMSE`;
   - `Correlation`;
   - `Confidence`;
   - оценку точности;
   - heatmap корреляции;
   - траекторию на DEM;
   - совпадение профилей высот.

## Алгоритмы

### Terrain Matching

Основной подход: сравнить профиль рельефа из NMEA с профилями, снятыми с DEM вдоль возможных прямых маршрутов.

### Grid Search

Перебираются:

```text
стартовые координаты X/Y
скорости 10..50 м/с
азимуты 0..359°
```

Для каждой гипотезы строится траектория в UTM-метрах, с DEM снимаются высоты, затем считается качество совпадения с NMEA.

### Coarse-to-Fine

Чтобы не перебирать всю карту с максимальной точностью сразу, поиск идет этапами:

1. грубый поиск по карте;
2. seed-sweep вокруг центра DEM;
3. уточнение вокруг лучших кандидатов;
4. микро- и nano-уточнение стартовой точки;
5. финальная проверка на полном профиле.

Если первичный результат слабый, включается более плотный повторный поиск.

### RMSE

Основной режим. Минимизирует ошибку между профилем NMEA и профилем DEM:

```text
меньше RMSE => лучше совпадение высот
```

### Correlation

Показывает совпадение формы профиля:

```text
correlation ближе к 1 => форма рельефа совпадает лучше
```

В интерфейсе строится полная heatmap correlation по направлениям `0..359°` и выбранному диапазону скоростей.

### Weighted Scoring

Дополнительный режим для сложных профилей, где много плоских или слабых участков.  
Точки с большим перепадом высоты и кривизной получают больший вес, потому что они лучше помогают привязаться к карте.

### Bridge Mode

Дополнительный режим для неинформативных фрагментов маршрута.  
Он использует уверенный участок профиля как опору, продолжает единую прямую траекторию и проверяет дальнейшее совпадение по DEM. Это не замена основному алгоритму, а страховочный режим для сложных данных.

### Confidence

Уверенность считается по контрасту лучшего совпадения относительно других гипотез.  
Низкий confidence означает, что похожих маршрутов на карте много или профиль слабо информативен.

### Flat Terrain Check

Если дисперсия восстановленного рельефа почти нулевая, координаты по одному радиовысотомеру определить надежно нельзя. В таком случае алгоритм помечает участок как плоский/неинформативный.

## Структура проекта

```text
data/
  map.tif                 основной DEM для демо
  map_hillshade.tif       картинка рельефа, не используется в алгоритме

outputs/                  локальные NMEA и результаты, не коммитятся

rust/
  terrain_nav_core/
    src/lib.rs            быстрый перебор гипотез, RMSE, correlation
    Cargo.toml

scripts/
  generate_test_data.py   CLI-генератор NMEA

src/terrain_nav/
  io/
    dem.py                чтение DEM и CRS
    nmea.py               парсинг NMEA
  core/
    search.py             основной пайплайн локализации
    rust_core.py          ctypes-обертка над Rust
    utm_raster.py         подготовка DEM в UTM
    sampling.py           интерполяция высот
    geometry.py           геометрия траектории
    matching.py           Python fallback для ошибок
    flat_gap.py           поиск неинформативных участков
    flat_gap_search.py    Bridge mode
  simulation/
    generator.py          генерация тестового полета по DEM
  models.py               структуры результата

web/
  app.py                  Streamlit-интерфейс

app.py                    точка входа
requirements.txt
README.md
```

## Что закрыто по ТЗ

- преобразование радиовысоты в профиль абсолютных высот рельефа;
- поиск скорости, азимута и координат `x/y`;
- расчет путевого вектора `Vx/Vy`;
- корреляционное сравнение профилей;
- heatmap коэффициентов корреляции по направлениям;
- наложение траектории на DEM;
- генерация NMEA в формате `$GPGGA`;
- адаптация к плоскому рельефу;
- фильтрация/сглаживание;
- самооценка точности;
- ускорение на Rust.

## Ограничения

- текущая модель предполагает прямолинейный равномерный участок полета;
- барометрическая высота считается постоянной `1500 м`;
- на полностью плоском рельефе один радиовысотомер не дает уникальной привязки;
- качество зависит от разрешения DEM и выразительности рельефа;
- это прототип для хакатона, не сертифицированная авиационная система.
