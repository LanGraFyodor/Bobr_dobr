# Полет вслепую

Веб-прототип резервной навигации воздушного судна без ГНСС.  
Решение определяет координаты, скорость и курс по цифровой карте рельефа `DEM` и данным радиовысотомера в формате `NMEA`.

## Суть

Радиовысотомер показывает расстояние от борта до земли.  
При известной барометрической высоте полета `1500 м` из него восстанавливается профиль рельефа под самолетом:

```text
высота рельефа = 1500 м - радиовысота
```

Дальше алгоритм ищет на DEM-карте прямую траекторию, профиль высот которой лучше всего совпадает с профилем из NMEA.

## Что реализовано

- чтение DEM-карты `data/map.tif` через `rasterio`;
- перевод координат карты из `lat/lon` в UTM-метры через `pyproj`;
- генерация тестового NMEA-файла радиовысотомера;
- парсинг NMEA-сообщений `$GPGGA`;
- восстановление профиля рельефа из радиовысотомера;
- поиск стартовых координат `x/y`, текущих координат, скорости и азимута;
- перебор гипотез: стартовая точка, скорость, направление;
- снятие высот с DEM вдоль предполагаемой траектории;
- интерполяция высот через `scipy`;
- расчет ошибки `RMSE` между профилем NMEA и профилем DEM;
- двухэтапный поиск: грубый проход по карте и уточнение вокруг лучших кандидатов;
- проверка плоского рельефа;
- сглаживание найденной траектории;
- **[Новое]** 2D Фильтр Калмана (модель постоянной скорости) для сглаживания вектора скорости и курса;
- **[Новое]** Архитектура скользящего окна (Sliding Window Tracking) для обработки длинных маршрутов;
- **[Новое]** Кинематическое прогнозирование полета над плоскими поверхностями (тундра/вода);
- **[Новое]** JIT-компиляция (Numba) тяжелых циклов поиска для работы на маломощных БВ;
- confidence-оценка результата;
- веб-интерфейс на `Streamlit`;
- визуализация heatmap ошибок, траектории на DEM и сравнения профилей высот.

## Как пользоваться

1. Оставить стандартную DEM-карту `data/map.tif` или загрузить свою `.tif`.
2. Загрузить NMEA-файл или нажать `Сгенерировать NMEA`.
3. Задать скорость, курс, длительность и шум для тестового полета.
4. Нажать `Запустить локализацию`.
5. Посмотреть результат:
   - скорость;
   - курс;
   - стартовые координаты;
   - текущие координаты;
   - RMSE;
   - confidence;
   - heatmap ошибок;
   - траекторию на карте;
   - график совпадения профилей.

## Алгоритмы

### Terrain Matching

Сопоставление профиля рельефа, восстановленного из радиовысотомера, с профилями рельефа на DEM-карте.

### Grid Search

Перебор возможных прямых траекторий:

```text
стартовая точка x/y
скорость
азимут
```

### Coarse-to-Fine Search

Сначала выполняется грубый поиск по карте, затем уточнение вокруг лучших кандидатов.

### RMSE

Основная метрика качества:

```text
меньше RMSE => лучше совпадение профилей
```

### DEM Interpolation

Высоты с карты снимаются интерполяцией, потому что траектория не обязана проходить точно через центры пикселей DEM.

### Flat Terrain Check

Если дисперсия профиля рельефа слишком маленькая, алгоритм считает участок неинформативным для привязки к карте.

## Ограничения прототипа

- предполагается прямолинейный равномерный участок полета;
- барометрическая высота считается постоянной: `1500 м`;
- качество результата зависит от выразительности рельефа;
- на плоской местности возможна неоднозначная локализация;
- основной критерий сейчас `RMSE`, корреляцию Pearson можно добавить следующим шагом;
- для промышленного применения нужна оптимизация и проверка на реальных данных.

## Гражданские сценарии применения

- доставка грузов в удаленные поселки;
- снабжение геологоразведочных лагерей;
- доставка оборудования на арктические метеостанции;
- полеты к высокогорным базам;
- резервная навигация БПЛА при деградации ГНСС;
- постполетный анализ маршрута.

## Статус

Рабочий хакатонный прототип.  
Показывает принцип резервной навигации: по радиовысотомеру и DEM можно восстановить траекторию, скорость, курс и координаты без использования GPS.
