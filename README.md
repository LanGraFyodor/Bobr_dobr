# Полет вслепую

Веб-прототип резервной навигации воздушного судна без ГНСС.
Решение определяет координаты, скорость и курс по цифровой карте рельефа `DEM` и данным радиовысотомера в формате `NMEA`.

## Суть

Радиовысотомер показывает расстояние от борта до земли.  
При известной барометрической высоте полета `1500 м` из него восстанавливается профиль рельефа под самолетом:

```text
высота рельефа = 1500 м - радиовысота
```

Дальше алгоритм ищет на DEM-карте прямую траекторию, профиль высот которой лучше всего совпадает с профилем из NMEA.

## Что реализовано

- чтение DEM-карты `data/map.tif` через `rasterio`;
- перевод координат карты из `lat/lon` в UTM-метры через `pyproj`;
- генерация тестового NMEA-файла радиовысотомера с отдельным seed маршрута и seed шума;
- парсинг NMEA-сообщений `$GPGGA`;
- восстановление профиля рельефа из радиовысотомера;
- поиск стартовых координат `x/y`, текущих координат, скорости и азимута;
- расчет текущего вектора скорости `Vx/Vy`;
- перебор гипотез: стартовая точка, скорость, направление;
- снятие высот с DEM вдоль предполагаемой траектории;
- интерполяция высот через `scipy`;
- расчет взвешенной ошибки `Weighted RMSE` между профилем NMEA и профилем DEM;
- расчет взвешенной корреляции профилей;
- основной score выбора результата: `RMSE` + штраф за слабую корреляцию;
- опциональный `Weighted scoring` для маршрутов с длинными плоскими участками;
- быстрый Rust-core для перебора траекторий;
- многоэтапный поиск: грубый проход, seed-sweep, уточнение, микро- и nano-уточнение;
- информативное прореживание профиля для быстрого грубого поиска;
- досрочное отбрасывание слабых гипотез в Rust-core;
- проверка плоского рельефа;
- фильтр Калмана и скользящее среднее для сглаживания траектории;
- confidence-оценка результата;
- эвристическая самооценка точности в метрах;
- веб-интерфейс на `Streamlit`;
- визуализация heatmap корреляции, траектории на DEM и сравнения профилей высот.

## Запуск

Установить зависимости:

```bash
pip install -r requirements.txt
```

Для быстрого ядра нужен установленный Rust toolchain (`cargo`). При первом запуске Python автоматически соберет Rust-библиотеку из `rust/terrain_nav_core`.

Запустить веб-приложение:

```bash
python -m streamlit run app.py
```

Открыть в браузере:

```text
http://localhost:8501
```

## Как пользоваться

1. Оставить стандартную DEM-карту `data/map.tif` или загрузить свою `.tif`.
2. Загрузить NMEA-файл или нажать `Сгенерировать NMEA`.
3. Задать скорость, курс, длительность и шум для тестового полета.
4. Нажать `Запустить локализацию`.
5. Посмотреть результат:
   - скорость;
   - курс;
   - вектор скорости `Vx/Vy`;
   - стартовые координаты;
   - текущие координаты;
   - Weighted RMSE;
   - correlation;
   - confidence;
   - оценку точности;
   - heatmap корреляции;
   - траекторию на карте;
   - график совпадения профилей.

## Алгоритмы

### Terrain Matching

Сопоставление профиля рельефа, восстановленного из радиовысотомера, с профилями рельефа на DEM-карте.

### Grid Search

Перебор возможных прямых траекторий:

```text
стартовая точка x/y
скорость
азимут
```

### Coarse-to-Fine Search

Сначала выполняется грубый поиск по карте, затем seed-sweep вокруг центра DEM, затем уточнение вокруг лучших кандидатов, микро- и nano-уточнение стартовой точки.
Если радиус поиска равен `0`, поиск идет по всей DEM-карте.

Для грубого прохода используется не весь профиль, а наиболее информативные точки: участки с сильным изменением высоты и кривизной рельефа. Финальная проверка выполняется на большем числе точек.

### RMSE и Weighted RMSE

Базовая метрика абсолютной ошибки высот:

```text
меньше RMSE => лучше совпадение профилей
```

Обычный `RMSE` используется как основной режим, потому что он стабильнее на большинстве тестов.

`Weighted RMSE` доступен как дополнительный режим для маршрутов, где большая часть профиля плоская, а полезная привязка находится на коротком участке гор/берега. В этом режиме вес точки растет на перепадах и кривизне рельефа.

### Weighted Pearson Correlation

Метрика совпадения формы профиля:

```text
correlation ближе к 1 => форма рельефа совпадает лучше
```

Финальный кандидат выбирается по комбинированному score: низкий `RMSE` важен, но при близких ошибках выигрывает профиль с более высокой корреляцией. Если включён `Weighted scoring`, вместо обычного `RMSE` используется `Weighted RMSE`.

### Confidence и оценка точности

Confidence считается по контрасту лучшего совпадения на heatmap.  
Оценка точности в метрах учитывает ошибку профиля, `correlation`, `confidence` и выразительность рельефа.

### Kalman Filter

Найденная траектория сглаживается фильтром Калмана с состоянием:

```text
x, y, vx, vy
```

### Rust Acceleration

Перебор гипотез выполняется в Rust в несколько потоков:

```text
start_x/start_y x speed x azimuth x profile_points
```

Это убирает основной тормоз Python-версии.

Rust-core досрочно отбрасывает траекторию, если по уже накопленной ошибке она не может обогнать текущих лучших кандидатов.

### DEM Interpolation

Высоты с карты снимаются интерполяцией, потому что траектория не обязана проходить точно через центры пикселей DEM.

### Flat Terrain Check

Если дисперсия профиля рельефа слишком маленькая, алгоритм считает участок неинформативным для привязки к карте.

## Ограничения прототипа

- предполагается прямолинейный равномерный участок полета;
- барометрическая высота считается постоянной: `1500 м`;
- качество результата зависит от выразительности рельефа;
- на плоской местности возможна неоднозначная локализация;
- для полного поиска по большой карте нужно задавать разумный район поиска;
- для промышленного применения нужна проверка на реальных данных и интеграция с другими датчиками.

## Гражданские сценарии применения

- доставка грузов в удаленные поселки;
- снабжение геологоразведочных лагерей;
- доставка оборудования на арктические метеостанции;
- полеты к высокогорным базам;
- резервная навигация БПЛА при деградации ГНСС;
- постполетный анализ маршрута.

## Статус

Рабочий хакатонный прототип.  
Показывает принцип резервной навигации: по радиовысотомеру и DEM можно восстановить траекторию, скорость, курс и координаты без использования GPS.
