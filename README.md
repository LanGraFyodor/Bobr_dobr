# Полёт вслепую

Прототип навигационного решения для легких беспилотных и опционально-пилотируемых воздушных судов, выполняющих доставку грузов в удаленные районы при отказах или деградации ГНСС.

Решение должно по цифровой модели рельефа и потоку данных простого радиовысотомера определить координаты воздушного судна, скорость и направление движения.

## Задача

Коммерческая авиакомпания эксплуатирует парк воздушных судов, которые летают над тайгой, тундрой, степями, горными плато и другими территориями без развитой наземной инфраструктуры.

Основной источник навигации, ГНСС, может быть недоступен или неточен из-за:

- неблагоприятной геометрии спутников в высоких широтах;
- ограниченного обзора неба в горах и глубоких долинах;
- спуфинга или джамминга рядом с объектами с мощным радиооборудованием;
- сильной облачности, геомагнитных бурь и других факторов.

Необходимо разработать алгоритм, который в реальном времени или постфактум восстанавливает:

- текущие координаты `x`, `y`;
- вектор скорости;
- путевой угол, азимут;
- путевую скорость.

## Входные данные

### Цифровая модель рельефа

Для алгоритма используется основной DEM-файл:

```text
map.tif
```

Файл `map_hillshade.tif` является только изображением рельефа и не должен использоваться в алгоритме.

В качестве источников DEM могут использоваться:

- Copernicus GLO-30 Digital Elevation Model;
- SRTM, Shuttle Radar Topography Mission;
- ALOS AW3D30.

### Данные радиовысотомера

Данные радиовысотомера генерируются самостоятельно и подаются на вход алгоритма в формате NMEA-0183 v3.

Пример сообщения:

```text
$GPGGA,123519.111,,,,,,,,545.4,M,46.9,M,,*47
```

В этом сообщении `545.4` - измеренная высота в метрах.

Параметры потока:

- частота сообщений: 1-10 Гц;
- барометрическая высота полета: 1500 м над уровнем моря.

## Основной алгоритм

1. Прочитать поток NMEA-сообщений радиовысотомера.
2. Преобразовать данные радиовысотомера в профиль абсолютных высот рельефа вдоль трассы полета.
3. Построить эталонные профили по DEM для возможных направлений из произвольной стартовой точки в центре карты.
4. Перебрать азимуты от `0` до `360` градусов с шагом `1` градус.
5. Найти угол и смещение, при которых коэффициент корреляции между измеренным и эталонным профилем максимален.
6. По найденной траектории вычислить координаты, скорость, путевой угол и путевую скорость.

## Чтение DEM

Функция чтения карты находится в `dem_reader.py`.

Она:

- читает `map.tif` через `rasterio`;
- возвращает 2D-массив высот;
- строит 2D-сетки координат пикселей;
- переводит координаты карты из `lat/lon` в UTM-метры через `pyproj`;
- автоматически выбирает UTM-зону по центру DEM.

Пример:

```python
from dem_reader import read_dem_as_utm

dem = read_dem_as_utm("map.tif")

heights = dem.heights
x_meters = dem.x_utm
y_meters = dem.y_utm

print(heights.shape)
print(dem.utm_crs)
```

Установка зависимостей:

```bash
pip install -r requirements.txt
```

## Визуализация

Программа должна строить:

- тепловую карту коэффициентов корреляции по всем направлениям;
- найденную траекторию поверх карты высот.

## Дополнительные улучшения

Возможные направления развития прототипа:

- фильтр Калмана для сглаживания шумов;
- самооценка точности результата;
- адаптация к плоскому или слабовыраженному рельефу;
- оптимизация времени расчета для маломощного бортового вычислителя;
- устойчивость к выбросам и шумам радиовысотомера.

## Бизнес-сценарии

Решение ориентировано исключительно на гражданское применение:

- доставка грузов в удаленные поселки и труднодоступные районы;
- обеспечение геологоразведочных лагерей;
- снабжение арктических метеостанций;
- доставка оборудования и медикаментов на высокогорные базы;
- постполетный анализ маршрутов при деградации навигационного сигнала.

Технология может быть установлена на легкие беспилотные и опционально-пилотируемые воздушные суда как дополнительный навигационный контур. На борту требуются радиовысотомер, барометрический высотомер, вычислительный модуль, локальное хранилище DEM и программный модуль корреляционной навигации.

## Структура данных

```text
map.tif             - основной DEM, используется алгоритмом
map_hillshade.tif   - визуализация рельефа, не используется алгоритмом
```
